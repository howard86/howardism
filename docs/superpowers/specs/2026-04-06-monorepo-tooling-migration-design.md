# Monorepo Tooling Migration Design

## Summary

Migrate the howardism monorepo from its current tooling stack to a modern, streamlined setup:

- **pnpm** -> **Bun** (package manager + workspace + runtime)
- **Lerna + Nx** -> **Turborepo + Changesets** (task runner + versioning/publishing)
- **ESLint + Prettier** -> **Ultracite** (linting + formatting via Biome)

Three sequential PRs, each fully completing one migration before the next begins.

## Current State

| Concern               | Current Tool                                                                                         |
| --------------------- | ---------------------------------------------------------------------------------------------------- |
| Package manager       | pnpm 8.15.9                                                                                          |
| Workspaces            | pnpm-workspace.yaml (`apps/**`, `packages/**`)                                                       |
| Task runner           | Lerna (delegates to Nx for caching)                                                                  |
| Task caching          | Nx 18.3.5                                                                                            |
| Build orchestration   | Turborepo (partially configured, `turbo.json` exists)                                                |
| Versioning/publishing | Lerna (independent versioning, gitmoji conventional commits, npm publish)                            |
| Linting               | ESLint 8 with `eslint-config-howardism` (airbnb, Next.js, React, import sort, jest, testing-library) |
| Formatting            | Prettier 3 (`printWidth: 100`, `singleQuote: false`, `semi: false`)                                  |
| Pre-commit            | Husky + lint-staged + commitlint (gitmoji)                                                           |
| CI                    | GitHub Actions: `test.yml` (lint/type-check/test via `nx affected`), `version.yml` (lerna publish)   |
| Runtime               | Node.js >= 18 (CI tests on 18 + 20)                                                                  |

## Design Decisions

1. **Bun as full runtime** where possible, falling back to Node for Next.js apps if compatibility issues arise.
2. **Changesets** for versioning/publishing (replacing Lerna's publish workflow).
3. **Ultracite presets as-is** -- adopt Ultracite's formatting opinions (2-space indent, 80-char width, semicolons) rather than replicating old Prettier settings.
4. **Full ESLint/Prettier replacement** -- accept loss of ESLint plugins with no Biome equivalent (testing-library, jest, Next.js plugin).
5. **Keep Husky + commitlint** (gitmoji), replace lint-staged with `biome check --staged --write`.
6. **Three sequential PRs** -- each independently reviewable and revertable.

## Phase 1: Bun Workspace Migration

### Changes

**Delete:**

- `pnpm-lock.yaml`
- `pnpm-workspace.yaml`

**Add:**

- `bun.lock` (generated by `bun install`)

**Update root `package.json`:**

- Remove `"packageManager": "pnpm@8.15.9"`
- Add workspaces field:
  ```json
  "workspaces": ["apps/*", "packages/*"]
  ```

**Update scripts:**

- All `pnpm run X` invocations become `bun run X`
- `pnpm install` becomes `bun install`

**CI (`test.yml`):**

- Replace `pnpm/action-setup` with `oven-sh/setup-bun`
- Replace `pnpm install` with `bun install`
- Replace `pnpm nx affected --target=X` with equivalent (Lerna/Nx still present this phase)
- Remove Node version matrix (18 + 20). Replace with single Bun version via `oven-sh/setup-bun`. Keep `actions/setup-node` with a single Node version only if Next.js builds require it (determined during implementation).

**CI (`version.yml`):**

- Same package manager swap (pnpm -> Bun)
- Lerna still runs here for now

**`workspace:*` references:** No change needed -- Bun supports this protocol natively.

### Risks

- Some dependency `postinstall` scripts may fail under Bun -- catch during `bun install`.
- Lerna/Nx may have issues under Bun -- invoke via `bunx` if needed.
- Next.js apps may need Node fallback for `next dev`/`next build`.

### Validation

- `bun install` succeeds with no errors
- `bun run build` completes for all packages
- `bun run lint`, `bun run test`, `bun run type-check` pass
- CI workflow runs green

## Phase 2: Turborepo + Changesets (replacing Lerna + Nx)

### Changes

**Delete:**

- `lerna.json`
- `nx.json`
- Dependencies from root `package.json`: `lerna`, `nx`, `conventional-changelog-gitmoji-config`
- `nrwl/nx-set-shas` step from CI

**Update `turbo.json`:**

```json
{
  "$schema": "https://turborepo.com/schema.json",
  "ui": "tui",
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "inputs": ["$TURBO_DEFAULT$", ".env*"],
      "outputs": ["dist/**", ".next/**", "!.next/cache/**"]
    },
    "lint": {},
    "type-check": {
      "dependsOn": ["^build"]
    },
    "test": {
      "dependsOn": ["^build"]
    },
    "clean": {
      "cache": false
    },
    "dev": {
      "cache": false,
      "persistent": true
    }
  }
}
```

**Update root `package.json` scripts:**

- `lerna run build` -> `turbo run build`
- `lerna run lint` -> `turbo run lint`
- `lerna run test --stream` -> `turbo run test`
- `lerna run type-check --stream` -> `turbo run type-check`
- `lerna run clean` -> `turbo run clean`

**Add Changesets:**

- Install: `@changesets/cli`, `@changesets/changelog-github`
- Create `.changeset/config.json`:
  ```json
  {
    "$schema": "https://unpkg.com/@changesets/config@3.1.1/schema.json",
    "changelog": ["@changesets/changelog-github", { "repo": "howard86/howardism" }],
    "commit": false,
    "fixed": [],
    "linked": [],
    "access": "public",
    "baseBranch": "master",
    "updateInternalDependencies": "patch"
  }
  ```

**Replace `version.yml` workflow:**
New workflow using Changesets:

- On push to master: run `changeset version` to bump versions, then `changeset publish` to publish to npm.
- Use `changesets/action` GitHub Action for the PR-based release flow.

**CI (`test.yml`):**

- Remove `nrwl/nx-set-shas` step
- Replace `pnpm nx affected --target=lint` -> `bunx turbo run lint`
- Replace `pnpm nx affected --target=type-check` -> `bunx turbo run type-check`
- Replace `pnpm nx affected --target=test` -> `bunx turbo run test`

### Trade-offs

- Lose Nx `affected` command. Turborepo caching achieves similar efficiency -- unchanged packages hit cache. For this repo size, the difference is negligible.

### Validation

- `turbo run build` completes for all packages
- `turbo run lint`, `turbo run test`, `turbo run type-check` pass
- Turborepo cache works (second run is near-instant)
- `changeset version` and `changeset publish` work correctly
- CI workflow runs green

## Phase 3: Ultracite (replacing ESLint + Prettier)

### Changes

**Delete:**

- `packages/eslint-config-howardism/` (entire package)
- Root `.eslintrc.js`
- All `.prettierrc*` files and `prettier.config.*` files
- Dependencies from root `package.json`: `eslint`, `eslint-config-howardism`, `prettier`, `prettier-plugin-tailwindcss`, `lint-staged`
- Any per-app ESLint configs

**Add:**

- `ultracite` as root dev dependency
- `biome.json` at repo root:
  ```json
  {
    "extends": ["ultracite/biome/core", "ultracite/biome/react", "ultracite/biome/next"]
  }
  ```

**Update scripts:**

- Root `lint` -> `turbo run lint` (each package's `lint` script becomes `biome check .`)
- Root `format` -> `biome check --write .`
- Per-app/package `lint` scripts -> `biome check .`

**Update Husky pre-commit hook:**

- Replace lint-staged invocation with `biome check --staged --write`
- Keep commitlint hook as-is

**Reformat codebase:**

- Run `biome check --write .` once to reformat everything
- Add the reformat commit SHA to `.git-blame-ignore-revs` so `git blame` skips it

### Accepted Trade-offs

- No equivalent for `eslint-plugin-testing-library` rules
- No equivalent for `eslint-plugin-jest` rules
- No equivalent for `@next/eslint-plugin-next` rules (Next.js still warns in `next dev`/`next build`)
- Formatting style changes to Ultracite defaults (2-space indent, 80-char width, semicolons, ES5 trailing commas)
- Import sorting handled by Biome's built-in organizer (replaces `eslint-plugin-simple-import-sort`)

### Validation

- `biome check .` passes with no errors
- `turbo run lint` passes across all packages
- Pre-commit hook works (`biome check --staged --write`)
- CI workflow runs green
- `.git-blame-ignore-revs` is configured

## Migration Order

```
Phase 1 (PR)          Phase 2 (PR)              Phase 3 (PR)
Bun workspace    ->   Turborepo + Changesets ->  Ultracite
                      (remove Lerna + Nx)        (remove ESLint + Prettier)
```

Each phase is a separate PR. A phase must be merged before starting the next.
