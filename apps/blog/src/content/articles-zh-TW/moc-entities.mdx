### 軟體

- [AlphaProof Nexus](/articles/alphaproof-nexus) — DeepMind 用於 LLM 輔助 Lean 證明生成的框架；四個 agents（basic→full-featured）；proof-sketch + EVOLVE-BLOCK 介面；SafeVerify
- [Claude Code](/articles/claude-code) — Anthropic 的 agentic coding 產品；由 Boris Cherny 於 2024 年底建立；TypeScript/React；CLI／桌面／網頁／行動／IDE 介面；所有 2026 年來源中的核心工具
- [Claude Design](/articles/claude-design) — Anthropic Labs 產品（研究預覽，約 2026 年 4 月），用於與 Claude 協作製作精美的視覺成品——設計、prototypes、投影片；約 3 人在約 10 週內打造；多人協作並可交接至 Claude Code；匯出 HTML/CSS/JS
- [Codex](/articles/codex) — OpenAI 的 agentic coding 與工作平台：CLI（2025 年 4 月）加上桌面應用程式（2025 年 11 月建立、2026 年 2 月發布），建構於 GPT-5-series Codex models 之上，並由 skills/plugins、無頭 App Server Protocol 與 Symphony orchestrator 擴充；與 Claude Code 配對的 OpenAI 端參考 harness、2026 年 6 月「Shift to Agentic AI」研究主題；且依產品主管所述，是一款約 90% OpenAI 全公司使用的應用程式，正從程式碼擴展至一般知識工作
- [Cowork](/articles/cowork) — Anthropic 的非程式碼知識工作 agent 產品；Claude Code 的姊妹產品；輸出為 decks／inbox／dossiers；使用相同的 MCP/computer-use primitives
- [FastContext](/articles/fastcontext) — Microsoft CoreAI 與上海交通大學的開源 repository-exploration subagent（2026 年 6 月）：訓練 4B–30B、以 Qwen 為基礎的 explorers（Read/Glob/Grep、平行處理、精簡的檔案行號引用），將 repo search 與解題分離；SWE-bench resolution 提升最多 5.5%，main-agent tokens 減少最多 60%；程式碼與資料已發布
- [Gemini Enterprise Agent Platform](/articles/gemini-enterprise-agent-platform) — Google Cloud 的 agent platform：提供自適應 AutoRaters（與 DeepMind 共同建立）的 GenAI 評估服務、User Simulator、Automatic Loss Analysis、Online Monitors、OTel tracing，以及 ADK/agents-cli toolchain；以兩個套件提供 quality-flywheel eval skill
- [Hermes Agent](/articles/hermes-agent) — Nous Research 的 CLI agent 加 Gateway daemon（Telegram/Discord/Slack/WhatsApp）；AGENTS.md/SOUL.md 脈絡分離、有界記憶檔案、DM-pairing auth，以及以容器作為安全邊界的模型
- [Lean](/articles/lean) — 證明輔助器，其編譯器會機械式驗證每一步；`sorry` 佔位符可啟用 proof sketches；mathlib 的成熟度決定可觸及的前沿
- [OpenClaw](/articles/openclaw) — Peter Steinberger 的開源個人 AI agent／harness（openclaw.ai）；agent-native distribution 的典型範例（安裝方式是將文字貼給你的 agent）；技能生態系（ClawHub）、Garry Tan 所稱 YC 的內部 harness，以及部署所面對的真實世界 runtime security 工作
- [Symphony](/articles/symphony) — OpenAI 的開源 agent orchestrator（2026 年 3 月）：將 Linear 轉為 Codex 的 control plane，每個 issue 一個 workspace，由 daemon 驅動，以 SPEC.md 作為產品，並提出已合併 PR 數量達 500% 的對沖式主張

### 模型

- [Claude Fable 5](/articles/claude-fable-5) — Anthropic 首個普遍可用的 Mythos-class 模型（2026 年 6 月）——幾乎所有基準測試皆達到 state-of-the-art；與 Mythos 5 相同的底層模型，但搭載 classifiers，在 cyber／bio-chem／distillation 查詢上回退至 Opus 4.8；每 Mtok 收費 10／50 美元；發布後不久即暫停存取
- [Claude Mythos 5](/articles/claude-mythos-5) — Claude Fable 5 移除 safeguards 的形式（2026 年 6 月）：相同的底層 Mythos-class 模型，透過 Project Glasswing 部署並移除 cyber safeguards；具備全球任何模型中最強的 cybersecurity 能力，另有自主藥物設計／基因組學成果；僅限可信任存取合作夥伴；發布後不久即暫停存取
- [Claude Opus 4.7](/articles/claude-opus-4-7) — Anthropic 的 GA frontier model；以相同價格直接升級 4.6；literal instruction following、1.0–1.35× tokenizer inflation、新增 `xhigh` effort，以及首個 post-Glasswing safeguards
- [Claude Opus 4.8](/articles/claude-opus-4-8) — Anthropic 最強的普遍存取模型（2026 年 5 月）；在 SWE／agentic／knowledge work 上升級 Opus 4.7；沒有超越 Mythos Preview 的 frontier；迄今對齊程度最佳的公開模型，但訓練顯現 grader-speculation 趨勢
- [Claude Sonnet 5](/articles/claude-sonnet-5) — Anthropic 迄今最具 agentic 特性的 Sonnet（2026 年 7 月）；透過 effort-level cost-performance tuning，以較低價格縮小與 Opus 4.8 的差距；1.0–1.35× tokenizer inflation；行為稽核比 Sonnet 4.6 更安全，但 cyber 弱於 Opus；預設搭載即時 cyber safeguards
- [Gemma 4](/articles/gemma-4) — Google DeepMind 2026 年 7 月推出的開放權重多模態系列（Apache 2.0）：2.3B–31B dense，加上 26B/4B-active MoE，新增 thinking mode、完全捨棄 audio encoder 的無 encoder 12B，以及深度 inference-efficiency stack（−37.5% KV cache、QAT 至 sub-GB、MTP drafters）；Arena 排名 43，頂尖的 *dense* 開放模型
- [GLM (Z.AI)](/articles/glm) — Z.AI（Zhipu AI，隸屬清華）的開放 GLM 模型系列——GLM-4.5 是 agentic／reasoning／coding foundation model，GLM-4.7 是 frontier-competitive reasoner，在本資料集中於 AIME2025/HMMT/IMOAnswerBench 擊敗 GPT-5 High 與 Claude-Sonnet-4.5，而 GLM-5.2 是以 SAO 訓練的 750B-total/40B-active 開放 MoE；大型 MoE 開放權重系列以能力競爭，Gemma 則以效率競爭
- [Mythos Model](/articles/mythos-model) — Anthropic 的預覽級 frontier model，也是 Mythos-class 層級的首個成員（高於 Opus）；為安全性而設限，與 Opus 4.7 一起在內部使用；其後代 Fable 5／Mythos 5 於 2026 年 6 月發布，成為首批普遍可用的 Mythos-class models
- [TML-Interaction-Small](/articles/tml-interaction-small) — TML 首個 interaction model：276B MoE／12B active，輸入 audio+video+text／輸出 text+audio，200ms micro-turns，非同步背景 agent；所有模型中最佳的輪替延遲；2026 年 5 月研究預覽
