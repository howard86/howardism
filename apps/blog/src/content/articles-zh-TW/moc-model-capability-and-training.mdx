> **model-capability-and-training** 領域的內容地圖 — 11 個概念。模型為何具備能力：測試時運算、能力過剩、RL 後訓練方法、推理效率，以及開放權重前沿。精選入口；所有領域請參閱[首頁](/)。


- [LLM 的非同步 RL](/articles/asynchronous-rl-for-llms) — 每個 rollout 一完成就立即用於訓練，而不是等待完整的同步批次 — 修正長尾 agentic／coding rollout 對 GPU 叢集造成的落後者閒置問題，但代價是 policy lag 與 off-policy drift；SAO 的 DIS（direct double-sided importance sampling）透過完全捨棄 old-policy model，並遮蔽任何 rollout 與目前機率比值離開嚴格 trust region 的 token，使其穩定下來
- [群組相對策略最佳化（GRPO）](/articles/group-relative-policy-optimization) — DeepSeek 的無 critic RL 目標函數，成為 2024–25 年 LLM 後訓練的預設方法：每個 prompt 取樣一組回應，以群組平均 reward 作為 baseline，並在沒有 value network 的情況下最佳化 clipped PPO surrogate — 在同步訓練中比 PPO 更便宜且穩定，但群組是一道隱含的同步屏障（更新必須等待最慢的成員），與非同步及單軌跡線上 agentic 設定不相容，這正是 SAO 所利用的落差
- [推理效率即能力](/articles/inference-efficiency-as-capability) — 如果能力是 inference budget 的函數，那麼降低 token 成本就是能力工作：Gemma 4 的五個槓桿（透過 keys-as-values + p-RoPE 將 KV-cache 減少 37.5%、把 QAT 壓到 sub-GB、MTP drafter heads、MoE、移除 encoder）讓每一美元能買到更多思考 — 也是 wiki 首次涵蓋部署端技術堆疊
- [鋸齒狀智慧（幽靈，而非動物）](/articles/jagged-intelligence) — 「幽靈而非動物」：鋸齒狀的統計電路，沒有內在動機；洗車／草莓失誤；保持在迴圈中，將其視為工具 — 而且跨越不同模型規模，推理會壓縮 10 倍，但儲存的知識不會
- [大規模測試時運算](/articles/large-scale-test-time-compute) *(hub)* — Noam Brown 的論點：模型能力如今是 inference budget（token／成本／時間）的函數；有良好 scaffolding 的現代模型能持續改進數週才達到平台期，因此不指明 budget，「模型有多大能力？」這個問題便無法妥善定義 — 這是造成基準測試、安全 evals 與快速起飛預測失效的根本原因
- [潛在能力過剩](/articles/latent-capability-overhang) — Noam Brown 主張，已發布的模型能做的遠比任何人已提取出的更多，因為沒有人投入足夠的測試時運算：OpenAI 以低成本反駁 Erdős unit distance conjecture，之後又透過 scaffolding（$1K–$100K）從 GPT-5.5 引導出相同結果；沒有人探索過將 $100K 的運算投入已發布模型能做到什麼；每次發布成本都下降 10–100 倍，助長「等下一個模型」的迷因
- [由 LLM 驅動的漏洞研究](/articles/llm-driven-vulnerability-research) — Claude Mythos Preview 湧現的資安能力：自主發現 zero-day、完成完整 exploit chain，以及 Anthropic 對此的 Project Glasswing 回應
- [開放權重前沿差距](/articles/open-weight-frontier-gap) — Arena Text，2026 年 6 月：頂尖 closed model 領先最佳 open model 33 Elo，領先最佳 *dense* open model 57；前沿的 open weights 意味著 744B–1.6T 的 MoE，因此 Gemma 4 31B 是在不同軸線上競爭（效率、edge deployment），而非縮小差距 — 而且 DeepMind 自己的 MoE 輸給了自己的 dense model
- [單次 Rollout 最佳化](/articles/single-rollout-optimization) — SAO 的核心做法：每個 prompt 只進行一次 rollout，而不是 GRPO 的群組，並在完成瞬間送入訓練 — 減少 off-policy drift，適用於每個 prompt 始終只提供一條 trajectory 的 online／agentic 設定；代價是類似 REINFORCE 的變異，因此必須重新採用 value model，以補償缺少 group-baseline，並將全部工程預算投入讓 critic 穩定（更快的 value updates、frozen-attention critic、skip-observation GAE、scaled value pretraining）
- [Software 3.0](/articles/software-3-0) — Karpathy 的分類法：1.0 是程式碼，2.0 是權重，3.0 是 prompting；LLM 作為可程式化的 interpreter；MenuGen「不應存在」；neural-net-as-host-process 外推
- [苦澀的教訓](/articles/the-bitter-lesson) — Sutton 2019：可擴展的一般方法勝過手工設計的結構；這是 wiki 中反覆出現、用來將 harnesses 溶入模型的理由；但有例外 — 機械驗證、character，以及 inference path 本身可能不會向內遷移
