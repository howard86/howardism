> **evals-and-benchmarks** 領域的內容地圖—11 個概念。衡量模型的科學：基準有效性、污染、飽和、LLM 評審，以及源自 production 的評估。精選入口；所有領域請見 [Home](/)。


- [基準污染與去污染](/articles/benchmark-contamination-decontamination) — Sun、Zhan 與 Gales（Cambridge、arXiv 2606.23313）：當測試樣本洩漏至訓練資料時，基準資料污染會膨脹回報的 LLM 分數，而現有去污染方法只依總體準確率評判，且大多需要乾淨的參考模型。兩項貢獻：(1) 樣本層級的評估框架—每個樣本的分布距離（去污染模型與未污染模型輸出分布之間的 D_KL 與 D*_L1）揭示一項根本脫節（paraphrase+permutation 將資料集層級的殘餘污染從 17.2→8.4 降低，但 D_KL 實際上上升 >13%）；(2) Uncertainty-Based Decontamination（UBD），不需要乾淨的參考模型，也不需要知道哪些樣本受到污染—受污染模型的 deep ensembles（僅批次排序不同的 5 個 LoRA seeds）會將記憶樣本暴露為高信心但高變異（epistemic/knowledge uncertainty），因為記憶對批次排序敏感。每個樣本的純量 α̂ ≈ 1−2σ（σ = 真實標籤機率的 ensemble std，且高於信心閾值 T_p）會驅動 UBD-Debiasing（事後抑制膨脹的真實標籤質量，按比例重新分配，不更新權重，僅限 MCQ）或 UBD-Unlearning（朝去偏後的 soft target 微調；α̂≈1 時不執行任何操作）。在 MMLU-Pro/MATH-MCQA 上，它帶來 >40–60% 的相對 D_KL 降幅，勝過 paraphrase/permutation，甚至勝過參考模型的 DeconIEP baseline；ensemble uncertainty 與 oracle contamination 的 PCC 為 0.8–0.9，而 log-probability 維持在 &lt;0.4
- [基準分數冗餘](/articles/benchmark-score-redundancy) — Zeng 與 Papailiopoulos（Microsoft Research、arXiv 2606.24020）：一個 84 模型 × 133 基準的公開分數矩陣（2,604 個儲存格，填入率 23.3%）實際上是 rank-2—模型在全部 133 個基準上的分數大致由兩個數字決定（兩項診斷一致：Soft-Impute completion 在 rank 2 時達到最低，且 top-2 SVD components 在每個完全觀測的子矩陣中解釋 >90% 的跨模型變異），呼應異質 frontier-era 矩陣上先前的 g-factor／「general-capability + provider-residual」發現。BenchPress（logit-space rank-2 bias-decomposed ALS matrix completion）在 100% coverage 下，將每個留出儲存格預測至 4.6 MedAE points；一組精簡的 5 基準探測集 \{GPQA-D, HLE, Codeforces, MMLU-Pro, ARC-AGI-1\} 可將模型完整 scorecard 還原至 3.93 points（低成本集合為 4.55），預測保留 ≥5-point pairwise model rankings 的 92.1%，而在嚴格時間截點下的新模型只需 5 個 seed scores 即可達到 4.83 points—因此你可以從少數探測推斷 scorecard 的大部分內容，而不必執行每個 eval，並搭配 hybrid ensemble-spread + matrix-support reliability layer 標記哪些預測值得信任；但分數可被推斷，不代表不需要基準，rank-2 幾何是 snapshot-specific，而矩陣正是 compute-control critique 所警告的同一個未受控公開網格
- [計算量受控的基準測試](/articles/compute-controlled-benchmarking) — Noam Brown 對單一數字「benchmark grid」的批評：它沒有控制 test-time compute，因此已經失效；修正方式是以 cost/token/time 為 x 軸繪製效能；benchmark-maxxing、留出的私有集合、讓網格持續存在的 Goodhart bad-equilibrium、為何 routing/consensus 必須在相同預算下評判，以及 Gemma 4 的標題表格作為實際範例，將 thinking model 與 non-thinking predecessor 進行基準比較
- [DRACO Benchmark](/articles/draco-benchmark) — Perplexity 對 100 個源自 production 的深度研究任務所做的基準測試（10 個領域、40 個國家），由 26 位專家的評分規準依 accuracy/completeness/objectivity/citation 評分；Perplexity Deep Research 在每個領域與軸向都領先，Claude Opus 4.6 是最強的非 Perplexity 系統，factual accuracy 是普遍的弱點
- [LLM 作為評審](/articles/llm-as-a-judge) — 使用一個 LLM 依據 criteria/rubrics 評分另一個 LLM 的輸出；DRACO 的 protocol 是每項標準以二元 MET/UNMET 加 justification 評判，再以權重聚合為 normalized score 與 pass rate；關鍵特性—不同 judge models 的排名保持穩定，但絕對數值會變動，而自適應的 per-case rubrics（Google 的 AutoRaters）雖能偵測失敗，卻會把失敗混合沖淡，因而需要針對變動中行為建立穩定的 custom metrics；一項涵蓋 21 位評審／約 541K 次判定的 audit 發現，原始 exact-match agreement 將經 chance-correction 的 reliability 高估 33–41pp（kappa deflation），而高 test-retest 可能掩蓋嚴重的 position bias，因此在用於 thresholded use 前，評審需要通過 chance-correction、bias 與跨基準驗證
- [LLM 評審驗證](/articles/llm-judge-validation) — UC Berkeley 對 21 位評審／9 家 provider／約 541K 次判定的 audit（Norman et al., 2026）：LLM-as-a-judge validation 系統性地不夠嚴謹—exact-match agreement 將經 chance-correction 的 κ 高估 33–41pp（kappa deflation，所有 judge 都一致），評審排名在不同基準間最多移動 14 位，而高 test-retest reliability 掩蓋嚴重的 position bias（一致性—偏差悖論）；其成果濃縮為 5 步驟的 Minimum Viable Validation Protocol
- [超越準確率飽和的衡量](/articles/measuring-beyond-accuracy-saturation) — Nadgir、Kapoor、… Narayanan（以 Princeton 為首、14 位作者、arXiv 2606.26158）：當基準的準確率飽和（頂尖 agents 在統計上無法區分）時，領域的直覺反應是淘汰並替換為更難的後繼者—但這會偏袒準確率，並捨棄另外六個可衡量軸向。他們以 CORE-Bench Hard（科學程式碼的 computational reproducibility）為案例研究，將 *accuracy saturation 與 benchmark saturation 脫鉤*：(1) 飽和本身會浮現較弱 agents 看不見的 construct-validity threats—透過 Docent 的 log analysis 找到 15 個 task-level errors 與 20 個可利用的 shortcuts，產生修正後的 CORE-Bench v1.1（39 個任務）及 OOD suite CORE-Bench OOD（19 個任務、新領域），但準確率仍飽和（頂尖 agent 100%，接下來四個並列約 97.4%）；(2) 在統計上無法區分的 agents，在 reliability（更準確的 agents 更一致；所有 agents 都嚴重缺乏信心，約 93% 通過但自述信心僅 32.1%，且沒有一個 agent 能在判斷自身正確與錯誤執行方面勝過隨機）、efficiency（GPT-5.3-Codex 比同等準確率的同儕便宜約 60%；token-cost 與 dollar-cost 呈現不同故事），以及 model-vs-scaffold contribution（某模型上的 scaffold 使準確率擺動約 44pp；同一模型上的兩個 scaffolds 在 31% 的任務上意見不一致，但 oracle router 達到 100%；direct fixes 以 95%/68% 勝過 rewrites）方面仍有明顯差異；(3) 一項小型隨機人類研究發現，agent collaboration 使 reproduction time 減少超過一半（2.11×，p≈0.002；這很可能仍低估，因為 25 次手動執行中有 5 次碰到 3 小時上限，而 collaborative run 沒有一次碰到）。論點：不要淘汰已飽和的基準—重新為它加上 instrumentation
- [源自 Production 的評估](/articles/production-sourced-evaluation) — 從去識別化的真實 production 使用情境，而非 synthetic 或手工撰寫的任務，建立基準；DRACO 的核心方法—以 difficulty-proxied sampling 取樣、移除 PII、augmentation、可自動更新並設置 human QA gate；representativeness 與 over-specification 的取捨；production traffic 作為 proprietary eval asset
- [無參考資料的評審過度給分](/articles/reference-free-judge-over-crediting) — Kranti 與 Vajjala（arXiv 2607.12885）：提示中參考答案的存在與位置，是 LLM 評審判定答案正確／錯誤的一階決定因素—在無參考設定中，評審會系統性地對錯誤答案過度給分，而加入參考答案會翻轉最多 85% 的判定（多數 NR→RV 翻轉是 CORRECT→INCORRECT 的過度給分撤回），此效果在 English/Arabic/Telugu 間都成立，但在低資源 Telugu 上嚴重得多；兩階段 calibration（C1 與 C2：評審能否分辨對錯）+ sensitivity（NR/RV/RC：參考答案的存在是否會翻轉判定）pipeline，可在無參考部署前診斷評審是否適用，而人類標註確認由參考答案驅動的更嚴格判定更符合人類（Gemma 的 agreement 從 NR→RV 約 0.34→0.85），因此無參考的絕對分數確實膨脹，而不只是不同—這是與 Norman et al. 正交的 reliability-without-validity 案例，研究對象是在 temperature 0 下的三個 open/open-weight judge models，任務為 binary QA
- [依規模而異的提示敏感度](/articles/scale-dependent-prompt-sensitivity) — Large models 因為 overthinking，在 7.7% 的標準基準上表現不如 small ones；brevity constraints 可恢復 26pp，並在 GSM8K/MMLU-STEM 上完全逆轉 hierarchy
- [任務時間範圍的擴展](/articles/task-time-horizon-scaling) — METR 對 AI 能夠獨立可靠完成的 task length 所做的衡量，大約每 4 個月翻倍（先前為每 7 個月）：Opus 3 約 4 分鐘（2024 年 3 月）→ Opus 4.6 約 12 小時（2026 年）→ 預計 2027 年達數週；與 benchmark saturation（SWE-bench、CORE-Bench）相互配合
