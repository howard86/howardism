## 摘要

這是來自 [Anthropic Labs](/articles/anthropic-labs) 的 product，讓你用 plain language 描述想要什麼，並與 Claude 協作建立精緻的視覺 artifacts，包括設計、prototypes、slides、one-pagers 等等。它由一個「開發期間多數時候是三個人」的團隊打造，從想法到 launch「大約十週」，並在 [Dan Carey](/articles/dan-carey) 2026 年 5 月 talk 前約一個月以 **research preview** 上線（約 2026 年 4 月）。它是幾個 wiki concepts 背後的具體例子：[Prototype Over PRD](/articles/prototype-over-prd)、[Compounding Loop Optimization](/articles/compounding-loop-optimization) 與 [Build for the Next Model](/articles/build-for-the-next-model)。

## 起源

團隊中的一位 designer **Nate**（先前是 [Claude Code](/articles/claude-code) 的 designer）在投入另一個 bet 時，用一個週末 hack 出第一個 prototype。它刻意做得很薄：**Agent SDK**、一個非常薄的 IDE wrapper，以及他已經在 Claude Code 裡使用的一個現有 skill。第一個 prototype 是「一個 terminal 和一個 browser，差不多就這樣」。發布到 Slack 後，團隊的反應（「這裡有潛力 / 這裡完全壞掉，請修」）變成前幾週的 roadmap。Carey 的 framing 是：99% 的價值不是來自那個 prototype，而是來自「十週的 iterating and shipping，以及每天和使用者談話」。

## 關鍵功能（以及各自移除的 friction）

多數功能一開始都是對團隊*自己* workflow 的修補（[Compounding Loop Optimization](/articles/compounding-loop-optimization)）：

- **Multiplayer** — 對同一個 design 進行 real-time co-editing。打造它是因為 teammates 原本要一個接一個把 prototypes 來回交接；launch 後，這成為 users 的*第一個*要求。
- **Handoff to [Claude Code](/articles/claude-code)** — 不必重新 export files，也不必重新輸入 conversation 的 context，就能把 design 推進 production。Launch 後 users 的*第二個*要求是：「那我要怎麼把這個放進 production？」
- **Codebase exploration / GitHub linking** — 用你的 brand，並建在你現有的 code 之上來 ship。
- **Open export** — designs 以 **HTML, CSS, JavaScript** export（不是封閉格式），反映「be an open tool」philosophy。
- **Self-hosting** — 團隊用「Claude Design to design Claude Design」，這是 [Dogfooding as Product Discipline](/articles/dogfooding-as-product-discipline) 最純粹的形式。

## Product philosophy

- **Lift the floor, not the ceiling.** 團隊打造過進階、pixel-level 的「power user」controls，少數 vocal testers 很喜歡，但「其他每個人都討厭」——令人困惑，而且「actively harmful」。他們在一週內把它們拔掉。教訓是：要做「a tool that lifts the level of craft for everybody, not just the ceiling on power users」（參見 [Vibe Coding vs. Agentic Engineering](/articles/vibe-coding-vs-agentic-engineering) 的「vibe coding raises the floor」）。
- **Openness for needs you'll never meet.** 因為有些 power users 想要 tool 不會支援的具體能力，所以 bet 是最大程度的 openness：HTML/CSS/JS export 加上 integrations，讓 specialized users 能把 designs 帶進自己的 tools。
- **Forward-looking integration claim (verify).** 截至 2026 年 5 月 talk，Carey 說「this week or next week」團隊會 ship 讓**任何 design tool 透過它們現有的 MCPs 與 Claude Design integrate** 的能力（見 [MCP and Computer Use](/articles/mcp-and-computer-use)）。這被表述為即將發生，而非已確認 shipped；請把它當作帶日期的 forward claim，等待 verification。

## Launch 與 iteration metrics

- **從週五 launch 到接下來的週一之間 shipped 62 項 improvements**，全部來自 launch-day user feedback（token efficiency、image handling、codebase exploration、near-instant exports）。Carey 說：這不是 all-nighter effort，而是「very natural」，因為團隊已經操練 daily loop 十週。
- **約一個月後將 token limits doubled**，回應喜歡它、並想更常使用它的 users。
- 早期 prototype 的 capability gaps 不是由 engineering 補上，而是因 [Opus 4.7](/articles/claude-opus-4-7) shipping 而關上——這是典型的 [Build for the Next Model](/articles/build-for-the-next-model) payoff。
