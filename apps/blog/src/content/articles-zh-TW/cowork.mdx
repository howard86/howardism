## 摘要

[Anthropic](/articles/anthropic) 的知識工作代理產品，是 [Claude Code](/articles/claude-code) 的姊妹產品。Claude Code 的目標是輸出為*程式碼*的工作；Cowork 則針對輸出為*非程式碼*的工作：簡報、收件匣分流、客戶檔案、發布文件與會議準備。產品由 [Cat Wu](/articles/cat-wu) 共同領導。

## 定位（Cat Wu 的說法）

- **[Claude Code](/articles/claude-code) (CLI)** — 一次性的程式設計任務；CLI 會率先取得新功能；「所有工具中最強大的工具」
- **Claude 桌面版／IDE 中的 Claude Code** — 需要預覽窗格的前端工作；更適合不熟悉終端機的非技術使用者
- **Claude Code 行動版／網頁版** — AFK 時啟動任務；在推出前是「缺少的產品」——人們會在戶外把筆電繫在手機上
- **Cowork** — 任何輸出不是程式碼的工作

## Cat Wu 如何使用它

連接 Google Calendar、Slack、Gmail、Google Drive——「脈絡越多越好」。一個範例工作流程是：晚上將草擬的大綱及相關 Twitter／發布頻道連結提供給 Cowork，隔天早上回來時，就能得到一份 20 頁、打磨完成的簡報，再透過一輪回饋進行迭代。系統會載入 Anthropic 的設計系統，因此簡報看起來就像由設計師製作的一樣。

## 人們為何使用它（根據 Cat Wu）

「Applied AI」團隊——內部 token 使用量僅次於工程團隊、排名第二——使用 Cowork 來處理：
- 客戶會議準備（「dossier」工作流程：整理昨天的問題、行動項目與 ETA 研究）
- 收件匣歸零／Slack 歸零分流
- 從 Salesforce + Gong + Slack 擷取發布計畫、內部文件與針對客戶製作的簡報

## 它如何連結到更大的 Anthropic 路線圖

- [Boris Cherny](/articles/boris-cherny) 將 Cowork 稱為整合通用 [MCP 與電腦操作](/articles/mcp-and-computer-use) 的場域——與 Claude AI 使用相同的 MCP 連接器，加上 Anthropic 在**電腦操作**方面的領先地位，能支援缺乏 MCP 的軟體（Opus 4.7 的表現「相當不錯」）。
- 「公司裡已經沒有任何地方還在手動撰寫程式碼」——Cowork 涵蓋這項說法中的非程式碼部分（SQL、簡報與營運文件全部由模型生成）。

## 在創辦人手冊中（2026 年 5 月）

[The Founder's Playbook](/articles/ai-native-startup-lifecycle) 將 Cowork 定位為 AI 原生新創在每個階段的**營運層**：

- **構想階段** — 自動化客戶聯繫與排程。Cowork 使用已驗證的目標客群輪廓研究潛在客戶、撰寫個人化聯繫內容，透過 MCP 連接 Gmail／Calendar、管理對話串，並執行第 7 天的後續跟進。
- **MVP 階段** — 回饋迴路的後勤工作：聯繫早期使用者、安排回饋會議、結構化收集錯誤與功能需求，以及每週彙整。
- **發布階段** — 在營運層取代創辦人的注意力：稽核創辦人的週期性任務，分類為自動化／委派／僅限創辦人處理，並為適合自動化的候選任務建立工作流程邏輯。PM 作業系統：衝刺排程、錯誤分流路由與每週指標彙整。
- **擴張階段** — 企業級營運：工單路由、升級工作流程、由產品變更觸發的文件更新、續約追蹤與報告節奏管理。此外還包括 GTM 戰術執行：內容管線、對外推廣序列、分析師簡報後勤與 CRM 資料維護。

這份手冊的說法是：Cowork + [Claude Code](/articles/claude-code) 結合後，能讓「小團隊擁有規模大得多的組織所具備的支援姿態」。手冊特別強調，Skills 是將創辦人週期性工作流程編碼化的介面。

## 關鍵引述

- (Cat Wu)「如果我正在打造某個輸出是程式碼的東西，我會使用 Claude Code、桌面版，或行動版的 Claude Code。如果輸出是任何不是程式碼的東西，我就會使用 Cowork。」
- (Boris Cherny)「我認為 Claude design 是一個非常好的例子。它今天已經相當不錯了。未來會變得好很多。」（在他回答願景時，指的是 Cowork 類產品。）

## 開放問題

- Cowork 的 harness 與 [Claude Code](/articles/claude-code) 的相比如何？兩者都提供 Skills、MCP、子代理——但非程式碼輸出的失效模式不同（沒有測試套件、沒有編譯器，也沒有可供審查的差異）。
- Cowork 類輸出的 eval 規範是什麼？Cat Wu 表示記憶功能從 evals 中獲益良多；但目前不清楚簡報品質如何衡量。
