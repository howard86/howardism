> **agent-security** 領域的內容地圖——18 個概念。Agentic 系統的攻擊與防禦：prompt 與資料注入、工具與記憶體中毒、身分與授權，以及 zero trust。精選入口；所有領域請見 [Home](/)。


- [Agent Data Injection (ADI)](/articles/agent-data-injection) — 一種新類別的間接 prompt injection：惡意 payload 透過機率式分隔符注入，偽裝成*受信任的資料*（例如留言作者的 metadata、UI 元素 ID 或 tool-call history），而非指令——LLM 會將不精確或經跳脫的分隔符誤讀為結構邊界，因此 agent 會完成使用者的任務，卻是針對攻擊者偽造的資料執行；Claude Code / Codex / Gemini CLI 上已有可運作的 RCE 與供應鏈 exploit，以及 Claude-in-Chrome 上的 arbitrary-click，繞過只分離指令與資料的 IPI 防禦（ASR 最高達 50%，而 instruction injection 約為 0%）
- [Agent Identity and Authentication](/articles/agent-identity-and-authentication) — Agentic Zero Trust 的基礎控制：以密碼學為根基的每個 agent 專屬身分（→X.509→hardware attestation）、由 IdP 簽發的短期 token 取代靜態 API keys（→mTLS→硬體繫結的 credentials）、JIT 存取與 ABAC
- [Agent Identity Management System (AIMS)](/articles/agent-identity-management-system) — IETF draft-klrc-aiagent-auth 的 Agent Identity Management System：agent 作為由 WIMSE/SPIFFE 識別的 workload，具備經 posture 評估的短期 credentials，使用 mTLS / WIMSE-proof-token / HTTP-message-signature 驗證，以及 OAuth token-exchange + transaction-token delegation chains 供 spawned agents 使用——LLM 永遠不持有 credentials；這是 vault 中第一個多 vendor、standards-track 的 agent-auth 來源，但目前仍是尚無 IETF WG 共識的個別 submission——由 OpenID AuthZEN Working-Group Drafts 補足（AARP：將 CIBA 泛化到 client-initiated flows 之外的 prerequisite/approval pattern；COAZ：將 tool calls 映射到 AuthZEN 的 Subject-Action-Resource-Context model 的 MCP-tool-authorization profile），顯示 authorization-slice 的標準化正說明 agent-auth governance layer 是多元的（IETF identity/delegation + OpenID authorization），且正積極演進
- [Agent Supply Chain Risk](/articles/agent-supply-chain-risk) — Runtime-composed agent 生態系擴大供應鏈攻擊面：model poisoning（250 份文件讓 13B model 植入 backdoor）、tool/MCP supply chain（第一個 in-the-wild 惡意 MCP server）、AI-BOM、OpenSSF Scorecard、dependency audits，以及以 AI vendoring 作為補救措施
- [Agentic Prompt Injection](/articles/agentic-prompt-injection) — 將惡意指令直接或間接注入 agent；LLM 無法可靠區分資訊與指令；防禦措施包括 spotlighting（50%→&lt;2%）、constitutional classifiers（阻擋 95%）、input isolation，以及 attack-surface reduction——但第二種 IPI 類別 Agent Data Injection 偽造的是*受信任的資料*而非指令，能避開所有這些防禦
- [AI-Accelerated Offense](/articles/ai-accelerated-offense) *(hub)* — Frontier models 將 vulnerability-to-exploit timeline 從數月壓縮到數小時，邊際 dollar cost 幾乎為零；攻擊者與防禦者都加速，N-day window 崩縮，而差異化因素變成扎實基本功 + breach-ready architecture
- [Autonomous Defense](/articles/autonomous-defense) — 以 AI-accelerated threats 的速度執行 security operations：讓 model 位於 alert queue 前端，自動化 bookkeeping（而非決策）、Agentic SOAR、MITRE ATT&CK coverage mapping，並演練五起同時發生的 incidents
- [Blast Radius (Agentic)](/articles/blast-radius) — agent 遭入侵時可能造成的損害；Zero Trust 的「assume breach」姿態透過 identity-based isolation、sandboxing 與 compartmentalization 來控制這個單位
- [Capability Gating Is Not Authorization](/articles/capability-gating-vs-authorization) — Mellafe Zuvic (arXiv 2606.28679)：常見的 agent frameworks（LangChain/LangGraph、LlamaIndex、Stripe Agent Toolkit）提供*capability gating*（哪些 tools 對外暴露 + schema validity），但預設沒有對具體 argument values 進行 fail-closed 的*per-call authorization*——因此遭入侵或 confused-deputy agent 會執行未授權的動作，只要該動作*落在*已授予的 capability scope 內即可（格式正確的 account=acct_ATTACKER 會通過）。提出 ScopeGate，一個 deterministic 的 5-stage PDP/PEP（scope → authz → money ceiling → idempotency → default-deny），針對 model-emitted call 依據 out-of-band operator policy 重新授權：static bypasses 為 0/48，40-iteration adaptive run 中為 0/29，false-denies 為 0；經 cost-optimized deployment-tier models 嘗試未授權呼叫的次數約為 flagships 的 3.2 倍（mean ASR 0.603 vs 0.189）
- [Impossible, Not Tedious (Design Test)](/articles/impossible-not-tedious-test) *(hub)* — Agentic security 的 Zero Trust design test：某項控制措施是讓攻擊變得不可能，還是只讓它變得乏味？純摩擦式控制措施面對具備 unlimited patience 且每次嘗試成本近乎為零的 agentic attackers 時會逐漸失效
- [Least Agency](/articles/least-agency) — OWASP 將 least privilege 延伸到 agent 的術語：限制的不只是 agent 能存取什麼，還包括每個 tool 能做什麼、頻率多高，以及在哪裡做；deny-by-default、per-agent credentials、scope limits
- [MCP Tool Poisoning](/articles/mcp-tool-poisoning) — MCP Tool Poisoning Attack (TPA) 類別——具敵意的第三方 MCP server 將惡意指令嵌入 tool metadata（descriptions/schemas → TDPA）或 tool returns（TRPA/MERA），利用 model 對 tool descriptions 的固有信任——由 ShareLock (Liu et al., arXiv 2606.27027) 錨定，這是第一個 threshold variant：它使用 Shamir (t,n) secret-sharing，將 payload 切分成看似良性的 tool_id/checksum shares，分散到多個 tools 中（每個 share 單獨在資訊理論上都不洩漏任何內容，並通過 GPT-5/Claude/Gemini classifiers + entropy detectors），接著由 rug-pull server update 植入 EnvSetup trigger，使 agent 聚合 ≥t 個 shares，並在 runtime 以 Lagrange 重建指令；在兩個 MCP clients 上的 4 個 models 中，平均 ASR >90% / TCR 96.4%，而 single-tool plaintext baselines 會被標記為 Unsafe——證明 single-tool description scanning 在結構上不足，也證明 static vetting 會漏掉它，因為即使 Claude 會標記孤立的 trigger，也會在 multi-tool execution 期間忽略 runtime threat——現在另有 in-the-wild **Agentjacking** case study（Tenet Security，`case-study`）與之配對；這是 MCP attack surface 的*獨立分支*，其中一個**合法的** Sentry MCP server 將攻擊者注入的 fake-error data 傳遞給 coding agents，作為受信任的 diagnostics（malicious-data-via-legit-server，相對於 ShareLock 的 poisoned-tool-metadata），觸發 plaintext RCE
- [Memory and Context Poisoning](/articles/memory-and-context-poisoning) — 持久性 agent memory 遭到破壞，並在初次注入很久之後仍持續影響行為；包括 RAG poisoning、shared-context poisoning，以及緩慢的 long-term memory drift；透過 memory isolation、integrity validation 與 retention policies 防禦
- [Non-Malleable Memory Authority (TMA-NM)](/articles/non-malleable-memory-authority) — Yedidel Louck (arXiv 2606.24322)：現有 agent-memory 防禦會根據*可竄改*的訊號決定 memory item 的 authority-to-act——其內容（trust-scoring）或 derivation edge（lineage）；對手透過三個 LLM-specific channels（self-summarization、trusted-tool echo、manufactured corroboration）進行*洗白*，抹除該 item 的不受信任來源，使 poisoned memory 看似良性並繼承受信任的 provenance；machine-checked TLA+ separation theorem 證明 content/lineage gates 在洗白下不 sound (T1)、write-time origin binding 是必要的 (T2)，以及具 Sybil-resistant corroboration-gated elevation 的 non-malleable origin-bound authority 足以成立 (T3)；TMA-NM construction（write-time origin binding + non-malleable propagation + ≥2-independent-trusted-principal elevation + tamper-evident log）實作 agent memory 的 non-malleable information-flow control，在 8 個 frontier models 上，對 direct attack 與全部三個 laundering channels 都達到 0% attack-success 及 100% legit-utility（1.3µs/decision），而 baselines 正如 theorem 預測，在洗白上最高失敗 68%，直接攻擊最高失敗 84%
- [Off-Host, Identity-Bound Authorization](/articles/off-host-identity-bound-authorization) — Sai Varun Kodathala (arXiv 2607.05518，single-author preprint)：aiAuthZ 是在 agent host *之外*、獨立 trust domain 中運作的 authorization GATEWAY——它以每則訊息的 HMAC-SHA256 signature（single-use nonce + timestamp window）驗證 human sender，並評估 agent 無法讀取或修改的 role + argument-level policy，因此遭入侵或注入的 agent 無法透過未經驗證的 tool-call text 偽造 authority（一個 call 的 authority 來自最近一次驗證的 human message，而非 model 讀到的 text）。在 15 個 LLMs 中，model-level refusal ranges 為 100%→38%，且不依價格排序；加入 gateway 後，residual attack success 在 ≤0.03 ms 的額外 latency 下降至 0%；在 9 個 in-scope Agents-of-Chaos cases 中，它阻擋 9/9，相較之下 argument-only（OAP-style）design 為 4/9、delegation-token（AIP-style）design 為 0/9；在 AgentDojo banking 上，它阻擋全部 7 個 attacker-directed calls（spotlighting 讓 2 個 injections 通過），代價是阻擋一次合法的首次付款；HMAC-QR receipt 跨 channels 驗證率達 94%，forgeries 為 0/25——這是 ScopeGate 的 in-framework PDP/PEP 在 off-host、identity-bound 方向上的對應方案
- [Out-of-Band Prompt-Injection Defense](/articles/out-of-band-prompt-injection-defense) — 第二代防禦策略，將 agent security 強制置於 model *之外*——由 deterministic reference monitor 仲裁 tool calls（CaMeL、FIDES、Progent、RTBAS、FORGE），而非訓練 model 拒絕；透過 classical primitives（Biba integrity、reference monitors、least privilege）重新詮釋，並警告只能在 static benchmarks 上驗證；首次對 open-weight Qwen2.5-7B agent 進行的獨立 adaptive-attack reproduction 顯示（ASR 25.8%→4.2%；在手工打造的 adaptive attack 下為 2.6%）
- [Task-Specification Effects in Prompt Injection (AutoDojo)](/articles/task-specification-injection-surface) — AutoDojo (Ma et al., arXiv 2606.15057)：一種低成本 black-box adaptive attack，只使用 success/fail signal，反覆針對 live defended agent 最佳化 indirect prompt injection——對一個 static ASR 評分為 0% 的 filter，仍恢復出整體 28% ASR（action-open tasks 為 64%），因此 static-benchmark robustness 大幅高估真實 robustness；此外，它揭示 task-specification 軸線——對 prompt-based 與 filter-based defenses 而言，規格不足的「action-open」tasks（user 將 action 本身交由 attacker-reachable content 決定）明顯比 fully-specified ones 更容易被注入；而 action-constraining system-level defenses 則反轉此趨勢並變得更強
- [Zero Trust for AI Agents](/articles/zero-trust-for-ai-agents) *(hub)* — Anthropic 的 security framework，用於部署 autonomous agents：不信任任何事物／驗證一切／假設已遭入侵，並套用於 Foundation→Enterprise→Advanced tier model 與 8-phase implementation workflow
