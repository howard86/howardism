**預定範圍：**跨供應商的 **markdown-as-control-plane** 模式——以 repo 版本控制的純文字檔
設定 agent 行為：`CLAUDE.md` / `AGENTS.md` / `SOUL.md` / `WORKFLOW.md` /
`SPEC.md` / `.cursorrules`。比較各供應商在角色切分（專案 vs 人格 vs 工作流 vs 規格）上的差異，
並論證此模式為何正收斂為新興標準。
