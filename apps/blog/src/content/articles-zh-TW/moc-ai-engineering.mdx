> **ai-engineering** 領域的內容地圖 — 35 個概念。精選入口；所有領域請見首頁。


- [Agent Harness Engineering](/articles/agent-harness-engineering) — 長時間運行 LLM agent 的腳手架模式：環境設計、漸進式脈絡揭露、機械式架構強制、agent 程式碼審查
- [Agent Identity and Authentication](/articles/agent-identity-and-authentication) — agentic Zero Trust 的基礎控制：以密碼學根植的 per-agent 身分（→X.509→硬體 attestation）、以 IdP 核發的短期 token 取代靜態 API key（→mTLS→硬體綁定憑證）、JIT 存取與 ABAC
- [Agent Loop Pattern](/articles/agent-loop-pattern) — `/loop`（cron 排程）與 Ralph Wiggum（耗盡待辦）迴圈作為次世代 agent 原語；AFK 執行、平行 fan-out、「迴圈即未來」
- [Agent-Native Infrastructure](/articles/agent-native-infrastructure) — 世界仍為人類而建，必須為 agent 重寫；「我該 copy-paste 什麼給 agent？」；感測器/致動器；agent 對 agent 的 representation
- [Agent Supply Chain Risk](/articles/agent-supply-chain-risk) — 執行期組合的 agent 生態系擴大供應鏈攻擊面：model poisoning（250 份文件即可後門化 13B 模型）、工具/MCP 供應鏈（首個野外惡意 MCP 伺服器）、AI-BOM、OpenSSF Scorecard、相依性稽核，以及以 AI vendoring 作為緩解
- [Agentic Prompt Injection](/articles/agentic-prompt-injection) — 直接與間接將惡意指令注入 agent；LLM 無法可靠區分資訊與指令；防禦包括 spotlighting（50%→&lt;2%）、constitutional classifiers（95% 攔截）、輸入隔離與攻擊面縮減
- [AI-Accelerated Offense](/articles/ai-accelerated-offense) *(hub)* — 前沿模型將漏洞到 exploit 的時間線從數月壓到數小時、邊際美元成本極低；攻防雙方皆加速、N-day 窗口塌縮，差異化在於紮實基礎與 breach-ready 架構
- [Autonomous Defense](/articles/autonomous-defense) — 以 AI 加速威脅的速度運行安全營運：讓模型站在告警佇列前端、自動化簿記（而非決策）、Agentic SOAR、MITRE ATT&CK 覆蓋映射，並演練五起同時事件
- [Blast Radius (Agentic)](/articles/blast-radius) — agent 遭入侵時的潛在損害；Zero Trust「假設 breach」姿態以身分隔離、沙箱與隔艙來限制的單元
- [Building Is Cheap, Arguing Is Expensive](/articles/building-is-cheap-arguing-is-expensive) — 「技術辯論中，code 勝出」：產生三個 PR 而非白板；prototype 優於 design doc；縮減 design doc
- [Claude Code Auto Mode](/articles/claude-code-auto-mode) — Claude Code 權限模式：分類器自動核准安全 tool call、阻擋高風險者；介於預設與 `--dangerously-skip-permissions` 之間
- [Claude Code Best Practices](/articles/claude-code-best-practices) *(hub)* — Anthropic 的 Claude Code 有效用法指南：脈絡管理、驗證驅動開發、explore→plan→code 流程、環境設定
- [Client-Side Agent Optimization](/articles/client-side-agent-optimization) — AgentOpt 將開發者可控的 agent 優化（per-role 模型、budget、routing）與伺服器端 serving 區分；combo 抽象；最佳/最差組合之間 13–32× 成本差距
- [Code as Source of Truth](/articles/code-as-source-of-truth) — 高 coding 吞吐下文件易過時；把 spec/skill check 進 repo；經 Claude onboard；spec-drift 驗證
- [Codex App Server Protocol](/articles/codex-app-server-protocol) — Codex 無頭 session 的 JSON-RPC stdio 協定：initialize/initialized/thread-start/turn-start 握手、延續 turn 重用 thread_id、動態 tool call 做 token 隔離的 tool 注入
- [Compute Allocator](/articles/compute-allocator) — 人類角色演進：決定什麼值得花 compute；約 1% 生成 token 上線、99% 是投入對齊/溝通的腳手架；豐裕心態
- [Context Window Smart Zone](/articles/context-window-smart-zone) *(hub)* — smart zone 與 dumb zone（Dex Hardy / Matt Pocock）：二次 attention 縮放、~100K 標記與 advertised context 無關；clear-and-restart > compaction；status-line token 計數為必備紀律
- [Deep Modules for Agents](/articles/deep-modules-for-agents) — Ousterhout 深/淺模組套用於 agent-friendly codebase；push 與 pull 指令傳遞；reviewer 在全新脈絡；Sandcastle 三 agent 模式
- [Design Concept Grilling](/articles/design-concept-grilling) *(hub)* — Matt Pocock 的 `grill-me` skill；在任何 plan 之前到達 Brooks「design concept」；對 specs-to-code 的制衡；PRD 為目的地文件、Kanban 為旅程文件
- [Disposable Micro-Apps](/articles/disposable-micro-apps) — 依任務拋棄的客製 UI 來編輯 plan（「micro-software on top of micro-software」）；copy-back-to-markdown；在豐裕心態下合理
- [Harness Shrinkage as Models Improve](/articles/harness-shrinkage-as-models-improve) *(hub)* — 每次模型釋出 prompt 腳手架縮減；Cat Wu 的 pruning 紀律；Boris Cherny「一年後一百行程式」；機械驗證仍承重
- [HTML as the New Markdown](/articles/html-as-the-new-markdown) — Thariq Shihipar 論點：模型變強後，千行 markdown plan 壓垮*人類*；HTML artifact（視覺、互動）讓人類留在迴路。面向模型的 harness 縮小，面向人類的 harness 成長
- [Impossible, Not Tedious (Design Test)](/articles/impossible-not-tedious-test) *(hub)* — agentic 安全的 Zero Trust 設計測試：控制是讓攻擊不可能，還是僅僅繁瑣？僅靠摩擦的控制，面對具無限耐心且每次嘗試成本趨近於零的 agentic 攻擊者時，效果會急劇衰退
- [Least Agency](/articles/least-agency) — OWASP 將最小權限延伸到 agent：不只限制 agent 能存取什麼，還限制每個 tool 能做什麼、多頻繁、何處；預設拒絕、per-agent 憑證、scope 限制
- [Living Design System](/articles/living-design-system) — 從 repo 萃取的 `design_system.html` 作為可攜、人機可讀的單一真相；component playground；橋接工程 ↔ 非技術利害關係人
- [LLM-as-Compiler Knowledge Base](/articles/llm-as-compiler-knowledge-base) — Karpathy 架構：LLM 漸進把原始文件編譯成持久互連 wiki，以 ingest→compile→query→lint 四階段管線取代 RAG
- [MCP and Computer Use](/articles/mcp-and-computer-use) — Anthropic 兩種互補連接器：MCP 做結構化程式存取（Salesforce/Drive/Gmail/Slack/Figma + 利基產業系統）；Computer Use 作無 MCP 時的 GUI 驅動兜底；Boris Cherny「對模型而言只是 token」
- [Memory and Context Poisoning](/articles/memory-and-context-poisoning) — 持久 agent 記憶遭腐化、在初次 injection 後很久仍影響行為；含 RAG poisoning、共享脈絡 poisoning、長期記憶慢漂移；以記憶隔離、完整性驗證、保留政策防禦
- [Outsource Your Thinking, Not Your Understanding](/articles/outsource-thinking-not-understanding) — 「可外包思考，不能外包理解」；理解為不可委派的人類瓶頸；知識庫作理解工具
- [Ticket-Driven Agent Orchestration](/articles/ticket-driven-agent-orchestration) — 讓 Symphony 運作的反轉：ticket 為工作單元（非 session/PR）、DAG 相依、agent 可擴展工作圖、「目標而非狀態轉換」
- [The Verifiability Thesis](/articles/verifiability-thesis) *(hub)* — LLM 自動化你能*驗證*的，如同電腦自動化你能*指定*的；RL 驗證獎勵 → 參差尖峰；「可驗證 + lab 在意」；終將一切可驗證化
- [Verification as the New Bottleneck](/articles/verification-as-the-new-bottleneck) *(hub)* — Fiona Fung：coding 不再是瓶頸 — 驗證、審查、維護才是；shift-left；TDD 失去稅負；PR 週期時間漏斗分析
- [Vertical Slice Tracer Bullets](/articles/vertical-slice-tracer-bullets) — Pragmatic Programmer tracer bullet 套用於 agent 任務分解；垂直切片 > 水平分層；帶阻塞邊的 Kanban 優於編號階段 plan
- [Vibe Coding vs. Agentic Engineering](/articles/vibe-coding-vs-agentic-engineering) — vibe coding 抬升下限（人人都能建）；agentic engineering 在加速同時守住品質；「&gt;10x 且持續擴大」；以大型專案招聘、非解題
- [Zero Trust for AI Agents](/articles/zero-trust-for-ai-agents) *(hub)* — Anthropic 部署自主 agent 的安全框架：不信任任何/驗證一切/假設 breach，貫穿 Foundation→Enterprise→Advanced 層級與 8 階段實作流程
