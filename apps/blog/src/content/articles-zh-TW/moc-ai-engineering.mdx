> **ai-engineering** 領域的內容地圖（Map of Content）— 36 個概念。精選入門；所有領域請見 [首頁](/)。


- [Agent Context Files](/articles/agent-context-files) — 跨廠商的「markdown 即控制平面」模式：以儲存庫版控的純文字（CLAUDE.md / AGENTS.md / SOUL.md / WORKFLOW.md / SPEC.md /.cursorrules）來設定代理行為，依角色拆分到 project / personality / workflow / spec 各層
- [Agent Harness Engineering](/articles/agent-harness-engineering) — 為長時間執行的 LLM 代理搭建鷹架的模式：環境設計、漸進式情境揭露、機械式架構強制、代理程式碼審查
- [Agent Identity and Authentication](/articles/agent-identity-and-authentication) — agentic Zero Trust 的基礎控制：以密碼學為根基的每代理身分（→X.509→硬體認證）、以短期 IdP 簽發的 token 取代靜態 API 金鑰（→mTLS→硬體綁定憑證）、JIT 存取與 ABAC
- [Agent Loop Pattern](/articles/agent-loop-pattern) — `/loop`（cron 排程）與 Ralph Wiggum（清空待辦）迴圈作為次世代代理原語；AFK 執行、平行扇出、「迴圈即未來」
- [Agent-Native Infrastructure](/articles/agent-native-infrastructure) — 世界仍是為人類打造的，必須為代理重寫；「我要複製貼上什麼給我的代理？」；感測器／致動器；代理對代理的表述
- [Agent Supply Chain Risk](/articles/agent-supply-chain-risk) — 執行期組合而成的代理生態擴大了供應鏈攻擊面：模型下毒（250 份文件就能在 13B 模型植入後門）、工具／MCP 供應鏈（首例現實世界中的惡意 MCP 伺服器）、AI-BOM、OpenSSF Scorecard、相依性稽核，以及以 AI vendoring 作為修補手段
- [Agentic Prompt Injection](/articles/agentic-prompt-injection) — 對代理直接與間接注入惡意指令；LLM 無法可靠地區分資訊與指令；防禦手段包括 spotlighting（50%→&lt;2%）、constitutional classifiers（攔截 95%）、輸入隔離，以及縮減攻擊面
- [AI-Accelerated Offense](/articles/ai-accelerated-offense) *(hub)* — 前沿模型以邊際的金錢成本，把「漏洞到 exploit」的時間軸從數月壓縮到數小時；攻防雙方同時加速、N-day 窗口崩塌，決勝差異變成扎實的基本功 + 已為被攻破做好準備的架構
- [Autonomous Defense](/articles/autonomous-defense) — 以 AI 加速威脅的速度來運作安全維運：把模型擺在告警佇列的最前端、自動化記帳工作（而非決策）、Agentic SOAR、MITRE ATT&CK 涵蓋率對應，以及演練五起同時發生的事件
- [Blast Radius (Agentic)](/articles/blast-radius) — 代理一旦被攻破可能造成的損害；這正是 Zero Trust 的「假設已被攻破」姿態所要遏制的單位，手段包括以身分為基礎的隔離、沙箱化與區隔化
- [Build for the Next Model](/articles/build-for-the-next-model) — 打造「快要能用」而非「已經能用」的原型：賭下一個具體的模型版本（而非遙遠未來的 AGI）會修好你的工程做不到的事；Claude Design 的 Opus 4.7 回報是最乾淨的案例
- [Building Is Cheap, Arguing Is Expensive](/articles/building-is-cheap-arguing-is-expensive) — 「在技術辯論中，程式碼會贏」：生成三個 PR 勝過在白板上爭論；原型勝過設計文件；減少設計文件
- [Claude Code Auto Mode](/articles/claude-code-auto-mode) — Claude Code 的權限模式，用分類器自動核准安全的工具呼叫並封鎖有風險的呼叫；介於預設模式與 `--dangerously-skip-permissions` 之間的折衷
- [Claude Code Best Practices](/articles/claude-code-best-practices) *(hub)* — Anthropic 關於高效使用 Claude Code 的指南：情境管理、驗證驅動開發、explore→plan→code 工作流程、環境設定
- [Client-Side Agent Optimization](/articles/client-side-agent-optimization) — AgentOpt 把開發者可控的代理最佳化（每角色選模型、預算、路由）框定為有別於伺服器端服務的事；「combo」抽象；最佳與最差組合之間 13–32× 的成本差距
- [Code as Source of Truth](/articles/code-as-source-of-truth) — 在高編碼吞吐量下文件會過時；把 specs／skills 簽入儲存庫；透過 Claude 上手；spec 漂移驗證
- [Codex App Server Protocol](/articles/codex-app-server-protocol) — 用於 headless Codex 工作階段的 JSON-RPC stdio 協定：initialize/initialized/thread-start/turn-start 握手、後續輪次重用 thread_id、以動態工具呼叫做 token 隔離的工具注入
- [Compute Allocator](/articles/compute-allocator) — 人類不斷演變的角色：決定什麼值得花費算力；生成的 token 約 1% 會出貨，99% 是投入於對齊／溝通的鷹架；豐盈心態
- [Context Window Smart Zone](/articles/context-window-smart-zone) *(hub)* — 智慧區 vs 笨拙區（Dex Hardy / Matt Pocock）：注意力呈二次方擴展、約 100K 的標記與宣稱的情境長度無關；清空重啟 > 壓縮；狀態列的 token 計數是不可或缺的紀律
- [Deep Modules for Agents](/articles/deep-modules-for-agents) — 把 Ousterhout 的深模組 vs 淺模組套用到對代理友善的程式碼庫；推送式 vs 拉取式的指令傳遞；在全新情境中的審查者；Sandcastle 三代理模式
- [Design Concept Grilling](/articles/design-concept-grilling) *(hub)* — Matt Pocock 的 `grill-me` skill；在任何計畫之前先抵達 Brooks 的「design concept」；對「規格轉程式碼」的反制；PRD 作為目的地文件、Kanban 作為旅程文件
- [Disposable Micro-Apps](/articles/disposable-micro-apps) — 為每個任務打造、用完即丟的客製 UI 以編輯計畫（「微軟體疊在微軟體之上」）；複製回 markdown；在豐盈心態下是合理的
- [Harness Shrinkage as Models Improve](/articles/harness-shrinkage-as-models-improve) *(hub)* — 每次模型發布，prompt 鷹架都會收縮；Cat Wu 的修剪紀律；Boris Cherny「一年後只剩 100 行程式碼」的主張；機械式驗證仍是承重結構
- [HTML as the New Markdown](/articles/html-as-the-new-markdown) — Thariq Shihipar 的論點：隨著模型進步，上千行的 markdown 計畫會淹沒*人類*；HTML artifact（視覺化、可互動）讓人類保持在迴路中。面向模型的 harness 收縮，而這個面向人類的 harness 則成長
- [Impossible, Not Tedious (Design Test)](/articles/impossible-not-tedious-test) *(hub)* — 針對 agentic 安全的 Zero Trust 設計測試：某項控制是讓攻擊變成不可能，還是只是繁瑣？只靠摩擦力的控制，面對有著無限耐心、每次嘗試成本近乎為零的 agentic 攻擊者時會失效
- [Least Agency](/articles/least-agency) — OWASP 把最小權限延伸到代理的術語：不只限制代理能存取什麼，還限制每個工具能做什麼、多常做、在哪裡做；預設拒絕、每代理憑證、範圍限制
- [Living Design System](/articles/living-design-system) — 從儲存庫抽取出的 `design_system.html`，作為可攜、人類與機器皆可讀的真實來源；元件遊樂場；橋接工程 ↔ 非技術利害關係人
- [LLM-as-Compiler Knowledge Base](/articles/llm-as-compiler-knowledge-base) — Karpathy 的架構：LLM 漸進式地把原始文件編譯成持久、互相連結的 wiki，以 4 階段的 ingest→compile→query→lint 管線取代 RAG
- [MCP and Computer Use](/articles/mcp-and-computer-use) — Anthropic 的兩種互補連接器機制：MCP 用於結構化的程式化存取（Salesforce/Drive/Gmail/Slack/Figma + 利基產業系統）；當沒有 MCP 時，computer use 作為驅動 GUI 的萬用方案；Boris Cherny 的「對模型而言，它只是 token」
- [Memory and Context Poisoning](/articles/memory-and-context-poisoning) — 持久性代理記憶的腐化，會在最初注入很久之後仍影響行為；包括 RAG 下毒、共享情境下毒，以及緩慢的長期記憶漂移；以記憶隔離、完整性驗證與保留政策來防禦
- [Outsource Your Thinking, Not Your Understanding](/articles/outsource-thinking-not-understanding) — 「你可以外包你的思考，但不能外包你的理解」；理解是無法委派的人類瓶頸；知識庫作為理解工具
- [Ticket-Driven Agent Orchestration](/articles/ticket-driven-agent-orchestration) — 讓 Symphony 得以運作的反轉：以 ticket 作為工作單位（而非工作階段／PR）、DAG 相依性、可由代理擴充的工作圖、「目標而非轉換」
- [The Verifiability Thesis](/articles/verifiability-thesis) *(hub)* — LLM 自動化你能*驗證*的事，正如電腦自動化你能*指定*的事；RL 驗證獎勵 → 參差的高峰；「可驗證 + 實驗室在意」；一切最終都可驗證
- [Verification as the New Bottleneck](/articles/verification-as-the-new-bottleneck) *(hub)* — Fiona Fung：編碼不再是瓶頸 — 驗證、審查、維護才是；shift-left；TDD 失去它的稅負；PR 週期時間漏斗分析
- [Vertical Slice Tracer Bullets](/articles/vertical-slice-tracer-bullets) — 把 Pragmatic-Programmer 的 tracer-bullet 模式套用到代理任務分解；垂直切片 > 水平分層；帶阻斷邊的 Kanban 勝過編號的階段式計畫
- [Vibe Coding vs. Agentic Engineering](/articles/vibe-coding-vs-agentic-engineering) — Vibe coding 抬高了下限（任何人都能建造）；agentic engineering 在加速的同時維持品質標準；「>10x 且持續拉大」；以大型專案而非謎題來招募
- [Zero Trust for AI Agents](/articles/zero-trust-for-ai-agents) *(hub)* — Anthropic 用於部署自主代理的安全框架：信任無物／驗證一切／假設已被攻破，套用在 Foundation→Enterprise→Advanced 的分級模型與 8 階段實作工作流程上
