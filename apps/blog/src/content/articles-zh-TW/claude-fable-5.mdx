## 摘要

Claude Fable 5 是 [Anthropic](/articles/anthropic) 首款**普遍開放使用的 [Mythos-class](/articles/mythos-model) model**（2026 年 6 月推出）—— 這是一個 Claude models 層級，在能力上「位於我們的 Opus class **之上**」。它被描述為在幾乎所有已測試的能力 benchmark（software engineering、knowledge work、vision、scientific research）上達到 state-of-the-art，而且「task 越長、越複雜」時領先幅度越大。Fable 5 是**與 [Claude Mythos 5](/articles/claude-mythos-5) 相同的底層模型** —— 兩者*只*在 safeguards 上不同（註：*Fable* 來自拉丁文 *fabula*，意為「被講述之物」，近似於希臘文 *mythos*）。Fable 啟用 safeguards 出貨；Mythos 則在某些領域解除它們。這套 safeguard architecture —— 由 classifiers 將 risky queries route 到 [Opus 4.8](/articles/claude-opus-4-8)，而不是拒答 —— 記錄在 [Capability-Gated Model Fallback](/articles/capability-gated-model-fallback)。

> **狀態（截至 2026-06-14 擷取版本）：access suspended。** 來源頁面現在最前方顯示 banner：Anthropic 正在暫停 Claude Fable 5 和 Claude Mythos 5 的 access，並表示很抱歉對客戶造成 disruption，正在努力盡快恢復 access。下方的 launch announcement 仍被保留；suspension 是即時狀態。

## 價格與 identity

- **每百萬 input tokens $10 / 每百萬 output tokens $50** —— 明確表示「不到 Claude Mythos Preview 價格的一半」。Fable 5 和 Mythos 5 採用相同價格。
- API model id：`claude-fable-5`（透過 Claude API / platform.claude.com）。
- Fable 5 的能力超過 Anthropic 曾經普遍開放使用的任何 model。

## 能力亮點

公告以 Fable 5（general access）開頭；science-heavy results 則在 [Mythos 5](/articles/claude-mythos-5)（bio safeguards lifted）下執行，並彙整於 [Autonomous Scientific Discovery](/articles/autonomous-scientific-discovery)。Fable 的 headline demonstrations：

- **Software engineering。** [Stripe](https://stripe.com/) 表示 Fable 5「把數月 engineering 壓縮成數天」—— 在一個 **5,000 萬行 Ruby codebase** 中，一天完成 codebase-wide migration，這項工作「否則需要整個團隊手工做超過兩個月」。在 Cognition 的 **FrontierCode** eval（在符合 production-codebase standards 的*同時*完成困難 coding tasks）上，Fable 5 在 frontier models 中分數最高，「即使在 medium effort」也是如此，而且比過去的 Claude models 更 token-efficient。
- **Knowledge work。** 在 [Hebbia](https://www.hebbia.com/) 的 Finance Benchmark 上，於 senior-level reasoning（document reasoning、chart/table interpretation、problem solving）取得所有 model 最高分；[IMC](https://www.imc.com/) 表示 Fable 5「幾乎全面拿下他們的 trading-analysis evaluations」。
- **Vision（新 SOTA）。** 能從細緻的 scientific figures 中擷取精確數字；也能**只靠 screenshots** 重建 web app 的 source code。關鍵是，它需要較少 scaffolding —— **先前的 Claude models 即使用 helper harnesses，也很難玩 Pokémon FireRed；但 Fable 5 用最小化、vision-only harness 就打贏 FireRed**（raw game screenshots，沒有 maps/navigation aids/game-state）。這是 [Harness Shrinkage as Models Improve](/articles/harness-shrinkage-as-models-improve) 的 flagship case。
- **Memory and long-context。** 在「數百萬 tokens」範圍內仍保持專注，並使用自己的 notes 改善輸出。在 *Slay the Spire* 中，file-based persistent memory 對 Fable 5 performance 的提升幅度，**比它對 Opus 4.8 的提升多 3 倍**，而且 Fable 抵達 final act 的頻率多 3 倍。
- **Autonomy。** Fable 5 和 Mythos 5 能比任何先前的 Claude models 更長時間地 autonomously 工作 —— 見 [Task Time-Horizon Scaling](/articles/task-time-horizon-scaling)。

## Safeguards（Fable 存在的原因）

Fable 是**有 safeguards 的** SKU。因為 Mythos-class capability 可能被 misuse 而造成嚴重傷害（尤其是 cybersecurity、biology），Fable 出貨時附有涵蓋 **cybersecurity、biology & chemistry，以及 distillation** 的 classifiers。當 classifier 觸發時，回應會由 [Opus 4.8](/articles/claude-opus-4-8) **處理**而不是拒答，並且會告知 user。Anthropic 以保守方式調校 classifiers（它們有時會攔到 harmless requests，在少於 5% 的 sessions 中觸發）；**超過 95% 的 Fable sessions 完全不涉及 fallback**，而在這些 sessions 中，Fable 的 performance 實際上與 Mythos 5 相同。完整說明見：[Capability-Gated Model Fallback](/articles/capability-gated-model-fallback)。

Alignment：[automated alignment assessment](/articles/automated-behavioral-audit) 發現 Mythos 5 的 misaligned behavior 很低，且與 Opus 4.8 相近；鑑於它們是相同的底層模型，Fable 5 的 alignment level 也會相近。目前所有 Mythos-class traffic 都適用 **30-day data-retention** 要求（僅供 safety 使用）。

## 可用性與 rollout

- Fable 5 當天即全面可用；從發布起即在 Claude API 和 consumption-based Enterprise plans 完全可用。
- subscription plans 採分階段 rollout：從發布起**到 6 月 22 日**，在 Pro/Max/Team/seat-based Enterprise 上免費；自 **6 月 23 日**起需要 usage credits；Anthropic 表示一旦 capacity 允許，目標是恢復 Fable 5 作為 subscription plans 的標準部分。
- Demand 預期非常高且難以預測 —— 這是分階段 access 的陳述理由。（即時狀態見上方 suspension note。）
