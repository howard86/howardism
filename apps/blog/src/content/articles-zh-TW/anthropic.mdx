## 摘要

AI 安全公司；Claude 模型家族的供應商。宣稱使命：「為全人類打造安全的 AGI。」最初相較 OpenAI 資金不足；據報導截至 2026 年 4 月 ARR 達 110 億美元，成長迅速。內部以出貨節奏聞名（見 [AI Native Product Cadence](/articles/ai-native-product-cadence)），以及一套培養跨領域通才的招聘與團隊設計哲學（見 [Engineer PM Convergence](/articles/engineer-pm-convergence)）。

## 產品

- **Claude API / Claude Developer Platform** — 模型 API，附帶託管式 agent 服務
- **[Claude Code](/articles/claude-code)** — agentic 程式開發產品
- **[Cowork](/articles/cowork)** — 非程式碼知識工作 agent
- **Claude AI** — 聊天產品（claude.ai）
- **Claude Desktop** — Mac/Windows 應用程式

## 2026 年來源中提及的模型

- **[Claude Opus 4.7](/articles/claude-opus-4-7)** — 正式發布的前沿模型（當前版本）
- **[Mythos Model](/articles/mythos-model)** — 內部使用的預覽級模型；能力被描述為足夠重大，以至於公開發布因安全考量而受限
- Sonnet 4.6 及更早版本 — 歷史參考點

## 內部結構（根據 [Cat Wu](/articles/cat-wu)）

- 跨團隊約 30–40 位 PM
- 團隊系列：研究-PM、Claude Developer Platform、Claude Code、企業、成長
- Mike Krieger（前 Instagram 創辦人）領導 Anthropic Labs 孵化器第二輪；曾在規模化階段主導產品面
- Amanda — Claude 的角色塑造工作（見 [Claude Character as Product](/articles/claude-character-as-product)）
- 「Applied AI」團隊 — 技術性市場推廣角色；token 使用量僅次於工程團隊

## 文化筆記

- 「Just do things」— 內部座右銘，歸因於 Cat Wu 等人；跨職能的預設態度
- 使命 > 產品優先級 — 使命是所有優先級衝突的最終裁決依據
- 招聘能在長期爬坡中維持能量的業界老手；偏好低自我意識、「擁抱混亂」的人
- 內部使用前沿模型（「dogfooding」）是強制性的；模型層內外使用相同模型（產品面介面領先）
- 「我們公司已經沒有任何手動撰寫的程式碼了。所有 SQL 都是由模型撰寫的。」— [Boris Cherny](/articles/boris-cherny)
- Claude 之間透過 Slack 對話已成為日常內部工作流程

## 重要事件

- **2024 年末** — Anthropic Labs 孵化器成立；產出 Claude Code、MCP、桌面應用程式；發布後解散
- **2025 年 5 月** — Opus 4 發布；[Claude Code](/articles/claude-code) 的 PMF 轉折點
- **2026 年 3 月** — [Claude Code](/articles/claude-code) 原始碼因發布 PR 中的人為疏失而外洩；流程已強化
- **2026 年** — OpenClaw 第三方存取受限；優先第一方訂閱
- **2026 年約 4 月** — [Claude Opus 4.7](/articles/claude-opus-4-7) 發布
- **2026 年** — [Mythos Model](/articles/mythos-model) 內部使用；對外僅限預覽
- **2026 年 5 月** — 「The Founder's Playbook」電子書出版（Anthropic Startups Program）；本 wiki 首次出現創辦人/新創領域內容（[AI-Native Startup Lifecycle](/articles/ai-native-startup-lifecycle)、[Founder as Agent Orchestrator](/articles/founder-as-agent-orchestrator)）
- **2026 年 5 月** — Claude Code Security 以限量測試版推出（程式碼庫掃描 + 針對性修補供人工審查）
