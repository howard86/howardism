## 摘要

[Anthropic](/articles/anthropic) 的 agentic 程式開發產品，由 [Boris Cherny](/articles/boris-cherny) 於 2024 年底在內部孵化器（Anthropic Labs）中創建。最初是自動補全替代方案；隨著 Sonnet 3.5 轉向 agentic 模式；在 PMF 前期約 6 個月，直到 Opus 4（2025 年 5 月）帶來轉折點；隨後在 Opus 4.5、Opus 4.6 和 Opus 4.7 持續突破。截至 2026 年 5 月，它涵蓋 CLI、桌面、網頁、行動裝置和 IDE（VS Code、JetBrains）——相同的產品介面，不同的進入點。

## 程式碼庫

根據 [Boris Cherny](/articles/boris-cherny)（2026 年 3 月因發布流程 PR 中的人為疏失而公開洩漏——隨後已加固，非惡意洩漏），其實作並不特別：TypeScript + React。這是刻意的選擇：在語言覆蓋率很重要的時期（2024 年底），這對模型來說「非常符合分佈」。

Boris 的主張：「一年後只需 100 行程式碼」——參見 [Harness Shrinkage as Models Improve](/articles/harness-shrinkage-as-models-improve) 了解方向，而非字面意義。

## 介面形式

- **CLI** — 主要介面，最先獲得新功能。「所有工具中最強大的」（Cat Wu）。
- **桌面應用程式** — 前端工作的預覽面板整合；對非技術使用者更友善。
- **IDE 擴充套件** — VS Code、JetBrains；根據 Boris 的受眾調查，使用比例明顯低於 CLI。
- **行動裝置 + 網頁** — 離開電腦時啟動任務；Boris 的日常工具。
- **姊妹產品：[Cowork](/articles/cowork)** — 相同的基礎元件，非程式碼產出。

## 重要功能

- `/loop` — Claude 排程的 cron 工作；[Agent Loop Pattern](/articles/agent-loop-pattern) 的主要原語
- `/powerup` — 引導使用者瀏覽超過 100 項功能的導覽
- Sub-agents — token 隔離的 context window，回報摘要；參見 [Context Window Smart Zone](/articles/context-window-smart-zone)
- Skills — 儲存庫中的 markdown 檔案，Claude 可按需取用；參見 [Deep Modules for Agents](/articles/deep-modules-for-agents) 中的 push/pull
- Auto mode — 基於分類器的權限管控；參見 [Claude Code Auto Mode](/articles/claude-code-auto-mode)
- Routines — 伺服器端的 `/loop`；筆電關閉後仍可存續
- Code review — 斜線指令 + 多 agent 審查模式，根據 Cat Wu 在 Opus 4.5/Opus 4.6 前後變得可靠
- **Claude Code Security** — 限量測試版（2026 年 5 月）；掃描程式碼庫中的安全漏洞並建議針對性修補供人工審查。能發現傳統方法遺漏的問題。定位為 MVP→Launch 過渡期（參見 [AI-Native Startup Lifecycle](/articles/ai-native-startup-lifecycle)）；明確表示*不能*取代人工審查者或合格的合規審查
